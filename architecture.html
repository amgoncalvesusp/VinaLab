Visão geral

VinaLab é interface gráfica desktop (PySide6) para AutoDock Vina 1.2.x, com suporte a múltiplas funções de pontuação, triagem em lote, visualização 3D e relatórios PDF. Bilíngue PT-BR/EN.

🧬 Docking molecular

Núcleo

Vina API Python + fallback CLI incluído, GNINA opcional, Vinardo, AD4. Rescoring com RTMScore e DeltaVinaXGB.

⚙️ Conversão

PDB · MOL2 · SDF · MOL · PDBQT

RDKit + Meeko (preferido) com fallback Open Babel. Cargas Gasteiger validadas antes do docking.

🧪 Triagem virtual

Modo Triagem

Pasta com múltiplos PDBQT, cada ligante processado em sequência, agrupamento por cluster, consenso Borda/Z-score.

📊 Análise interativa

Resultados

Tabela filtrável, viewer 3D embutido, plot de consenso 2D/3D, exportação Excel/CSV/PDB complexo.

📄 Relatórios

ReportLab

PDF com parâmetros, gráfico, top poses, interações. Resumo automatizado para arquivo de projeto.

🌐 Bilíngue

PT-BR + EN

Alternância em tempo real via menu Idioma. Persistência em user_prefs.json.

Filosofia. Cada tab numerada representa uma etapa do workflow científico. Conversor (1) → Preparar Proteína (2) → Docking (3) → Relatório (4). Resultados visíveis em painel lateral fixo.

Módulos core

core/docking_engine.py

QThread Threaded Executa jobs Vina fora da GUI thread.

  • DockingWorker(QThread) — emite progress_signal, log_signal, result_signal, error_signal.
  • VINA_SCORING_NAMES — mapeia vina/vinardo/ad4/gnina.
  • EXTERNAL_SCORING_ARCHIVES — pacotes em pontuacao/: DeltaVinaRF20, DeltaVinaXGB-Light, RTMScore.
  • _ensure_pdbqt_charges() — chamado antes do Vina; valida coluna 9 e repara via RDKit/OBabel se inválida.
  • parse_output_pdbqt() — extrai REMARK VINA RESULT, CNNscore, CNNaffinity.

core/converter.py

Pipeline determinístico RDKit + Meeko com fallback OBabel:

# Ligante
RDKit (MolFromMol2File|MolFromPDBFile|SDMolSupplier)
  ↳ fallback MOL2: _mol2_fallback_via_molblock (extrai @<TRIPOS>ATOM/BOND -> V2000)
  ↳ Sanitize + Kekulize + AddHs(addCoords=True)
  ↳ ComputeGasteigerCharges
  ↳ Meeko MoleculePreparation.prepare(mol)
  ↳ PDBQTWriterLegacy.write_string()
  ↳ validate_ligand_pdbqt + _validate_ligand_bond_geometry

# Receptor
PDB -> mk_prepare_receptor.py (se disponível)
       ↳ fallback obabel -xr
MOL2 -> obabel mol2->pdb -> pipeline acima

core/file_utils.py

  • sanitize_pdbqt_for_vina(input, output_dir, role) — remove tags incompatíveis (BRANCH/ROOT em receptor, etc.), normaliza tipos atômicos AutoDock.
  • validate_ligand_pdbqt(path) — atom_count > 0, sem múltiplos MODEL, componente conexo único.
  • validate_pdbqt_charges(filepath) -> bool — preflight: cada ATOM/HETATM com float parseável na coluna de carga.
  • repair_pdbqt_charges(filepath, role) — RDKit Gasteiger → Open Babel --partialcharge gasteiger → fill 0.000.
  • pdbqt_coordinate_bounds(), pdbqt_coordinates(), pdbqt_receptor_atoms() — geometria.

core/i18n.py

Registro central I18n.STRINGS com chave → {"pt":..., "en":...}. Métodos: get(key, lang), load_lang(prefs_path), save_lang(lang, prefs_path). Persistência em config/user_prefs.json.

core/environment_manager.py

Detecta dependências obrigatórias (vina/RDKit/meeko) + opcionais (GNINA, RTMScore deps). Gera status_report cacheado consumido pelo botão "Corrigir agora".

Validação de cargas (Issue 21)

AutoDock Vina exige que a coluna 9 do registro ATOM/HETATM contenha float parseável (carga parcial Gasteiger).

Bug histórico. Ligantes convertidos sem cargas válidas faziam o Vina abortar com PDBQT parsing error: Charge "A " is not valid.

Estratégia de reparo

  1. RDKit Gasteiger — Strip dos tokens AutoDock (HD, OA, NA…) para um PDB temporário (colunas 1-66), aplica SanitizeMol + ComputeGasteigerCharges, reescreve coluna 9 preservando tipos atômicos originais.
  2. Open Babel — obabel in.pdbqt -O out.pdbqt --partialcharge gasteiger, copia cargas re-geradas para o arquivo original.
  3. Fallback 0.000 — Para receptores e ligantes onde as duas estratégias falharam. Vina aceita o arquivo, mas a precisão energética degrada.
# API exposta
from core.file_utils import validate_pdbqt_charges, repair_pdbqt_charges

if not validate_pdbqt_charges(path):
    repair_pdbqt_charges(path, role="ligand")   # ou "receptor"/"rigid_receptor"/"flex_receptor"

Preparar Proteína

Novo em 0.0.2 Pré-processamento de PDB antes da conversão para PDBQT.

Controles

  • Carregar PDB... — QFileDialog.getOpenFileName, popula cadeias detectadas.
  • Remover HETATM — exclui ligantes/águas/cofatores; campo "Manter resíduos" (vírgula) preserva ex. HEM, MG, ZN.
  • Cadeia — combo populado por parsing da coluna 22 do PDB.
  • Adicionar hidrogênios — chama obabel -h; para protonação dependente de pKa use H++ ou Reduce externamente.
  • Preparar e salvar — escreve PDB filtrado e emite receptor_prepared.

Encadeamento

PrepareProteinTab.receptor_prepared(filepath)
  └─ MainWindow._use_prepared_receptor()
       ├─ converter_tab.input_paths = [filepath]
       ├─ converter_tab.type_combo.setCurrentIndex(1)  # Receptor
       └─ tabs.setCurrentWidget(converter_tab)

Conversor

Aceita *.pdbqt *.mol2 *.sdf *.mol *.pdb. Detecta formato e mostra na console de log.

Pipeline ligante

flowchart LR A[Arquivo de entrada] --> B{_detect_format} B -- pdbqt --> X[Copia direto] B -- mol2 --> C[Chem.MolFromMol2File] C -- None --> D[_mol2_fallback_via_molblock] C --> E[Sanitize + Kekulize] D --> E B -- pdb --> F[Chem.MolFromPDBFile] F --> E B -- sdf --> G[SDMolSupplier] G --> E E --> H[AddHs + ETKDGv3 se sem 3D] H --> I[ComputeGasteigerCharges] I --> J[Meeko.prepare PDBQT] J --> K[validate_ligand_pdbqt]

Pipeline receptor

  • PDB → mk_prepare_receptor.py → fallback obabel -xr.
  • MOL2 → obabel mol2→pdb intermediário → mesmo pipeline acima.
  • PDBQT → cópia direta após detecção.

DockingTab (etapas 3-7)

3. Função de Pontuação — seleção múltipla

Cada função selecionada gera execução de validação separada. Ranking baseado em literatura 2021-2025 (vide Funções de pontuação).

4. Caixa de busca — center + size
  • X/Y/Z center — spinboxes -9999..9999 Å, passo 0.5
  • X/Y/Z size — 1..126 Å (limite Vina)
  • Predefinição de tamanho — Pequena/Média/Grande cubo
  • Ajustar tamanho pelo ligante — cubo = max(extensão) + 1 Å
  • Modo blind docking (Rg) — caixa = 2·Rg + 2 Å centralizada no centroide
  • Centro no átomo selecionado — árvore de resíduos/átomos do receptor
  • Centro no ligante de referência (co-cristal)
  • Salvar / Carregar predefinição... — JSON com center+size
5. Parâmetros do AutoDock Vina
  • exhaustiveness 1-512 (padrão 8)
  • num_modes 1-20 (padrão 9)
  • energy_range 1-10 kcal/mol
  • cpu 0=auto
  • seed + checkbox "Fixar semente"
  • min_rmsd separação mínima entre poses
6. Diretório de saída

Pasta onde Vina escreve *_out.pdbqt, logs, tabelas e validation_redocking/.

7. Checklist pré-execução

Status colorido por linha: receptor OK, ligante OK, sobreposição ligante/caixa, OpenBabel, função de pontuação disponível, executável Vina.

Workflow guiado

1

Preparar a proteína

Tab Preparar Proteína → Carregar PDB → marcar "Remover HETATM" (mantenha cofatores via lista de resíduos) → selecionar cadeia se houver múltiplas → opcionalmente "Adicionar hidrogênios" → Preparar e salvar. Resultado é encaminhado ao Conversor automaticamente.

2

Converter para PDBQT

Tab Conversor → arquivo já pré-preenchido (Receptor selecionado) → confirma saída → Converter para PDBQT. Repita para ligante(s) escolhendo "Ligante" no combo. Use Usar como receptor/ligante no Setup para enviar diretamente.

3

Configurar Setup

Tab Docking → painel 1. Configuração do receptor: confirme arquivo. Se for usar receptor rígido total, marque "Usar receptor totalmente rígido". Em 2. Configuração de ligantes escolha "Ligante único" ou "Modo Triagem (pasta)".

4

Definir scoring + caixa

Em 3. Função de Pontuação marque uma ou mais funções. Em 4. Caixa de busca use: (a) Centro no ligante de referência para co-cristais, (b) Modo blind docking (Rg) sem informação prévia, ou (c) coordenadas manuais. Ajuste tamanho via predefinição Média/Grande ou "Ajustar TAMANHO pelo ligante".

5

Parâmetros e execução

Em 5. Parâmetros ajuste exhaustiveness (mais alto = mais tempo, mais amostragem). Em 6. Diretório de saída escolha pasta. Confira 7. Checklist pré-execução: todos OK? Clique Executar docking.

6

Analisar resultados

Painel lateral popula em tempo real. Filtre por afinidade/RMSD. Use sub-tab Consensus Plot para comparar funções 2D/3D. Em Clusters ajuste cutoff RMSD. Em Interações selecione pose para ver resíduos do bolsão.

7

Validar e exportar

Menu Análise → Validar protocolo: redocking contra pose cristalográfica de referência, top-N de sucesso. Tab Relatório → Gerar PDF. Tabela → Excel com formatação. Complexo → exportar PDB combinado.

Multi-scoring. Marcar múltiplas funções gera execução paralela de validação. ResultsTab calcula consenso por Borda Count, Z-score e DP entre ranks.

Metalo-complexos (Issue 4)

Vina/Vinardo tratam metais como Lennard-Jones — sem termo de coordenação. AD4 é preferível para metaloproteínas via affinity maps customizados. Para Zn proteases ou heme Fe considere rescoring QM ou MetalDock externo.

Solução de problemas

Erro: PDBQT parsing error: Charge "A " is not valid

Causado por coluna 9 sem float Gasteiger. Desde v0.0.2 é detectado e reparado automaticamente por repair_pdbqt_charges(). Se o erro persistir, abra o arquivo manualmente e verifique se a saída do conversor terminou. RDKit + Meeko deve estar instalado para a melhor reparação.

Vina aborta com "ligand atoms outside box"

O checklist mostra "Coordenadas do ligante não sobrepõem a caixa". Use Centro no ligante de referência ou Modo blind docking para autocentrar. Aumente size via predefinição Grande ou "Ajustar TAMANHO pelo ligante".

RDKit não interpreta MOL2

Desde v0.0.2 há fallback _mol2_fallback_via_molblock que reconstrói V2000 a partir das tabelas @<TRIPOS>ATOM/BOND. Se falhar mesmo assim, converta para SDF/PDB externamente: obabel arquivo.mol2 -O arquivo.sdf.

GNINA indisponível

Coloque gnina.exe em tools/gnina/ ou no PATH. No Windows é prático rodar GNINA dentro de WSL e expor o binário. Sem GNINA, use RTMScore para qualidade equivalente em CPU.

Dependências opcionais ausentes (RTMScore/DXGB)

pip install torch dgl MDAnalysis ProDy torch-scatter. Linha do checklist fica amarela e a função permanece desabilitada se faltar qualquer uma.

Open Babel não encontrado

Win: pip install openbabel-wheel instala obabel.exe dentro do Scripts. Para protonação em Preparar Proteína é obrigatório.

Caixa de busca ultrapassa 126 Å

Limite hard do Vina 1.2.x. Spinboxes truncam automaticamente. Para targets grandes, faça blind docking em duas etapas ou divida em sub-grades.

FAQ

Qual a diferença entre Vina e Vinardo?

Vinardo é re-parametrização da função de pontuação Vina (Quiroga & Villarreal, PLoS ONE 2016). Supera Vina em todos os benchmarks CASF. Mesma velocidade.

Posso rodar múltiplos ligantes em paralelo?

Não. DockingWorker processa sequencialmente para evitar contenção de I/O e respeitar cpu. Múltiplas funções de pontuação dentro do mesmo ligante rodam em sequência também — o paralelismo é interno do Vina via --cpu.

Como salvo um projeto?

VinaLab não tem ".projeto". Salve manualmente: PDBQT inputs, vina_config.txt (botão Salvar configuração) e a pasta de saída. Predefinições da caixa via Salvar caixa → JSON em config/box_presets.json.

RMSD lb vs ub?

Lower bound (lb): RMSD considerando apenas átomos mais próximos. Upper bound (ub): RMSD com atribuição 1:1 ordenada. Diferenças grandes lb/ub indicam degeneração de simetria interna.

Posso usar GROMACS/Schrödinger output?

Sim, desde que seja convertido para PDBQT via Meeko/Open Babel. Use Preparar Proteína para limpar HETATM/águas antes.

O que persiste em user_prefs.json?

Idioma escolhido, flag de primeira execução, função(ões) de pontuação selecionadas. Apagar o arquivo restaura defaults.
